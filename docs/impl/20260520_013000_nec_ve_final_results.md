## 修复历程摘要（跨6次迭代）

| 问题 | 修复方案 |
|------|---------|
| `common.sh` 循环自源 | 删除自身 source 语句 |
| `sudo vecmd/lspci` 无权限 | 改用 sysfs / systemctl is-active |
| VE 节点编号 0→1,2,3 | 全局 VE_NODES="1 2 3" |
| `run_test()` 内嵌重定向冲突 | 删除所有 cmd 字符串内的 `>` 重定向 |
| AVEO libnfort_m.so 检测 | 直接文件存在性检查（非 ldconfig） |
| AVEO 运行时库路径 | `VE_LD_LIBRARY_PATH=/opt/nec/ve/nfort/5.4.1/lib` |
| MPI mpirun 路径 | `bin64/runtime/mpirun`，source necmpivars-runtime.sh |
| MPI Ping-Pong 节点对语法 | `-ve A,B` → MPMD: `-np 1 -ve A ... : -np 1 -ve B` |
| 压力测试负载过小 | ve_matmul N: 512 → 4096 |
| perf_latency 硬编码节点 | 改为从 VE_NODES 动态生成配对 |
